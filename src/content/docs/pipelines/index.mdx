# Pipeline syntax

Bld pipelines are built with backwards compatibility in mind — after a bld update, existing pipelines will not break even if new functionality is introduced or the pipeline structure changes altogether.

This is achieved through a version number that is separate from the release version of bld. When a breaking change is introduced in the pipeline syntax to support new features, the pipeline version is incremented. All existing pipelines continue to execute with the feature set they were designed for. This lets you migrate to a new pipeline syntax at your leisure.

The current available versions are **Version 1** and **Version 2** (latest).
