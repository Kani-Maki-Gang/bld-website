# Runs On

The `runs_on` field defines where pipeline commands execute. Three platforms are supported:

<Tabs>
  <Tab label="Machine">
    <Callout type="note" title="Version Support">
      Supported in `version 1` and `version 2`.
    </Callout>

    Commands execute directly on the host machine. Bld creates a temporary directory at `.bld/tmp/<uuid>` for the run, which is cleaned up afterward.

    ```yaml
    runs_on: machine
    ```
  </Tab>
  <Tab label="Container">
    Commands execute inside a Docker container. Several configuration styles are supported:

    <Tabs>
      <Tab label="Existing Image">
        <Callout type="note" title="Version Support">
          Supported in `version 1` and `version 2`.
        </Callout>

        Creates a container from an image that must already exist locally. You are responsible for maintaining the image.

        ```yaml
        runs_on: ubuntu
        ```
      </Tab>
      <Tab label="Pull Image">
        <Callout type="note" title="Version Support">
          Supported in `version 2` only.
        </Callout>

        Pulls the image if it is missing, then creates a container. Set `pull: true` to enable automatic pulling.

        ```yaml
        runs_on:
          image: ubuntu
          pull: true
        ```

        Pull a specific tag:

        ```yaml
        runs_on:
          image: ubuntu:20.04
          pull: true
        ```

        Specify a Docker URL from your configuration:

        ```yaml
        runs_on:
          image: ubuntu:20.04
          pull: true
          docker_url: unix_socket_docker_url
        ```

        Use a registry defined in your configuration or inline:

        ```yaml
        # Registry from global config
        runs_on:
          image: ubuntu:20.04
          pull: true
          registry: ghcr

        # Inline registry configuration
        runs_on:
          image: ubuntu:20.04
          pull: true
          registry:
            url: ghcr.io
            username: some_username
            password: some_password
        ```
      </Tab>
      <Tab label="Dockerfile">
        <Callout type="note" title="Version Support">
          Supported in `version 2` only.
        </Callout>

        Builds a custom image from a Dockerfile before creating the container.

        ```yaml
        runs_on:
          dockerfile: /path/to/dockerfile
          name: custom-image
          tag: latest
        ```

        With a specific Docker URL:

        ```yaml
        runs_on:
          dockerfile: /path/to/dockerfile
          name: custom-image
          tag: latest
          docker_url: unix_socket_docker_url
        ```
      </Tab>
    </Tabs>
  </Tab>
  <Tab label="SSH">
    <Callout type="note" title="Version Support">
      Supported in `version 2` only.
    </Callout>

    Commands execute on a remote host over SSH. Required fields: `host`, `user`, `userauth`. The `port` field is optional and defaults to `22`.

    You can also reference a named SSH profile from your `.bld/config.yaml`:

    ```yaml
    runs_on:
      ssh_config: my_global_ssh_config_key
    ```

    Three authentication methods are available:

    <Tabs>
      <Tab label="Password">
        ```yaml
        runs_on:
          host: 127.0.0.1
          port: 22
          user: some_user
          userauth:
            type: password
            password: the_user_password
        ```
      </Tab>
      <Tab label="Keys">
        ```yaml
        runs_on:
          host: 127.0.0.1
          user: some_user
          userauth:
            type: keys
            public_key: /path/to/public_key
            private_key: /path/to/private_key
        ```

        `private_key` is required. `public_key` is optional.
      </Tab>
      <Tab label="Agent">
        ```yaml
        runs_on:
          host: 127.0.0.1
          user: some_user
          userauth:
            type: agent
        ```

        Uses the local SSH agent for authentication.
      </Tab>
    </Tabs>

    See the [SSH server configuration](/docs/server/ssh/) page for details on defining global SSH profiles.
  </Tab>
</Tabs>
