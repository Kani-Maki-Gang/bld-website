# Variables

<Callout type="note" title="Version Support">
  Variables are supported in pipeline versions `1` and `2`. The syntax differs between versions — both are shown throughout this page.
</Callout>

The `variables` section is a hashmap where each key is a variable name and the value is its default. Variables can be overridden at runtime with the `-v` flag on `bld run`.

## Basic usage

Define a variable and reference it in a step:

<Tabs>
  <Tab label="Version 2">
    ```yaml
    name: Example pipeline with variables 1
    runs_on: machine
    version: 2

    variables:
      directory: /home/user/target_directory

    jobs:
      main:
      - ls ${{directory}}
    ```
  </Tab>
  <Tab label="Version 1">
    ```yaml
    name: Example pipeline with variables 1
    runs_on: machine
    version: 1

    variables:
      directory: /home/user/target_directory

    steps:
    - name: List all content of target_directory
      exec:
      - ls bld:var:directory
    ```
  </Tab>
</Tabs>

Variable syntax: `${{name}}` in version 2, `bld:var:name` in version 1.

Override the value at runtime with the `-v` flag:

```bash
bld run -p example.yaml -v directory=/home/kvl/another_directory
```

## Multiple variables

<Tabs>
  <Tab label="Version 2">
    ```yaml
    name: Example pipeline with variables 2
    runs_on: machine
    version: 2

    variables:
      first_directory: /home/user/target_directory
      second_directory: /home/user/another_directory
      third_directory: /home/user/yet_another_directory

    jobs:
      main:
      - ls ${{first_directory}}
      - ls ${{second_directory}}
      - ls ${{third_directory}}
    ```
  </Tab>
  <Tab label="Version 1">
    ```yaml
    name: Example pipeline with variables 2
    runs_on: machine
    version: 1

    variables:
      first_directory: /home/user/target_directory
      second_directory: /home/user/another_directory
      third_directory: /home/user/yet_another_directory

    steps:
    - name: List content of all directories
      exec:
      - ls bld:var:first_directory
      - ls bld:var:second_directory
      - ls bld:var:third_directory
    ```
  </Tab>
</Tabs>

Override multiple variables at runtime:

```bash
bld run -p example.yaml \
  -v first_directory=some_path \
  -v second_directory=some_other_path \
  -v third_directory=some_yet_another_path
```

<Callout type="note">
  Variable default values support expressions — you can reference other variables, environment variables, or built-in keywords as a default value.
</Callout>
